== Projet

**Nom** : mp3-organizer — gestionnaire de collection MP3 avec PostgreSQL + LLM
**Stack** : Kotlin, Gradle (application plugin), PostgreSQL (R2DBC), JAudiotagger, LangChain4j, Ollama
**DAG** : hors DAG (application standalone)
**Documentation** : AsciiDoc

=== Objectif

Scanner des répertoires MP3, extraire les tags ID3 (artiste, album, titre, genre),
stocker dans PostgreSQL, exporter en JSON/YAML/XML/SQL, organiser les fichiers par artiste,
générer des playlists VLC, et interagir en langage naturel via LLM local (Ollama).

=== Contexte — Cercles de Confiance

Ce projet vit dans `foundry/public/newpipe-gradle/mp3-organizer/` → *cercle 4 (public)*.
- Ne pas référencer de secrets ici (ils sont dans `configuration/`, cercle 1)
- Voir `configuration/.agents/AGENT_GOVERNANCE.adoc` pour l'ontologie spatiale complète

=== Composants Clés

* **Database.kt** : Connexion R2DBC PostgreSQL
* **Mp3Scanner.kt** : Scan récursif, extraction tags ID3 (JAudiotagger)
* **Export*.kt** : Export JSON/YAML/XML/SQL
* **OrganizeFiles.kt** : Organisation par artiste
* **ai/** : 10 classes LangChain4j + Ollama (SQL generation, chat, maintenance, playlists)
* **playlist/XspfGenerator.kt** : Génération playlists VLC
* **01_schema.sql** : PostgreSQL — tables, vues, fonctions stockées

== Session Procedure

=== Début de Session

1. Lire `PROMPT_REPRISE.adoc`
2. Lire `.agents/INDEX.adoc`
3. `git status`
4. `./gradlew compileKotlin`

=== Fin de Session — 6 étapes obligatoires

1. Rapport de session → `.agents/sessions/{N}-{sujet}.adoc`
2. Mise à jour `PROMPT_REPRISE.adoc` (max 70 lignes)
3. Mise à jour `INDEX.adoc`
4. Mise à jour `SESSIONS_HISTORY.adoc`
5. Mise à jour `TEST_COVERAGE_ANALYSIS.adoc` (si tests modifiés)
6. Mise à jour `COMPLETED_TASKS_ARCHIVE.adoc`
